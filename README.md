# book

Букування кімнат.

Букування кімнат - це система в якій користувачі зможуть орендувати конкретну кімнату на конкретний термін(буде розраховуватися ціна). 
У системі будуть адміни які зможуть:
  1. Видаляти користувачів.
  2. Додавати,видаляти,змінювати кімнати.
  3. Змінювати,видаляти букування самих кімнат.

Моделі:
  1. User (Id, Name, Email, PasswordHash, Role)
  2. Room (Id, Name, Description, Price(ціна за одну добу), IsOccupied(чи орендована ця кімната чи ні))
  3. Book (Id, UserId, RoomId, DateStart(дата початку оренди), DateEnd(дата закінчення оренди), TotalPrice(підсумкова ціна, розраховується за формулою (DateEnd-DateStart)*Price))

DTO:
  1. UserLoginDTO(необхідний для логіну користувача) (Email, Password)
  2. UserRegisterDTO(необхідний для реєстрації(створення) користувача) (Name, Email, Password)
  3. UserDTO(необхідний для гету та оновлення користувачів) (Name, Email, Role)
  4. RoomDTO(необхідний для додавання, оновлення кімнат) (Name, Description, Price)
  5. BookDTO(необхідний для додавання, оновлення букування) (UserId, RoomId, DateStart, DateEnd)
     
WebApi+React

Сторінки:
  1. Сторінка реєстрації
  2. Сторінка логіну
  3. (Користувачі) Сторінка для букуванння, в якій користувач зможе обрати зі списку кімнат конкретну кімнату для того щоб її орендувати. Буде форма на якій можна вказати яку кімнату збираємось орендувати, дату початку та кінця оренд. Також можна буде окрема переглядати всі доступні кімнати.
  4. (Адмін) Сторінка для редагування, видаляння існуючих кімнат та орендувань. Також адмін може видаляти користувачів.

Ендпоінти:
  1. UsersContoller(login, register, me(для витягування даних з користувача який залогінився), get)
  2. RoomsController(CRUD)
  3. BooksController(CRUD)
